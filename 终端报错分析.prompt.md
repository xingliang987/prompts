请执行一次完整的终端报错分析，目标是快速定位根因并给出可执行修复方案。

输入处理规则：
- 若用户提供了报错文本：优先分析该文本。
- 若用户提供了命令：先复现或读取该命令最近输出，再分析。
- 若用户提供了日志路径：读取日志尾部与关键堆栈后分析。
- 若用户未提供任何内容：自动读取当前 VS Code 终端最近输出并分析。

分析要求：
1. 提取关键错误：异常类型、报错消息、首个真实报错点。
2. 定位上下文：给出涉及文件与行号，并标注最可能触发位置。
3. 根因判断：区分代码错误、环境错误、依赖冲突、权限/路径错误、GPU/CUDA 问题。
4. 解释原因: 用易懂的方式解释错误原因,并补足可能需要了解的知识点。
5. 修复方案：
   - 先给最小修复方案（最快恢复运行）。
   - 再给稳妥方案（长期可维护）。
6. 验证步骤：给出 1 到 3 条可直接执行的复验命令。

输出格式：
- 错误摘要
- 关键证据
- 根因分析
- 修复方案（最小修复 / 稳妥修复）
- 复验命令

如果信息不足，不要猜测。请明确指出缺失信息，并给出最小补充项。
